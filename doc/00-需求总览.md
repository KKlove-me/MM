# 家庭物资管理系统需求总览

## 1. 产品定位

家庭物资管理系统用于管理普通家庭中的食品、日用品、园艺用品等常用物资。系统采用本地数据库 + C/S 桌面端形态，第一版不考虑登录、权限、多用户协作或云端同步。

系统重点解决以下问题：

- 家里有什么物资、还有多少、放在哪里。
- 哪些物资已开封或正在消耗。
- 哪些物资临期、过期或需要尽快处理。
- 哪些物资消耗较快，需要提前补货。
- 同一种物资即使品牌、包装、批次不同，也能按同一口径统计。

## 2. MVP 范围

第一版包含：

- 物资分类和基础资料管理。
- 库存批次录入、编辑、查询和状态管理。
- 存放位置管理。
- 独立消耗单管理，支持一次性消耗、部分消耗和持续消耗。
- 已开封/消耗中物资统计。
- 库存、分类、位置、临期、过期等基础统计。
- 基于历史消耗速度的轻量预测和补货建议。
- 本地数据库存储。

第一版不包含：

- 用户登录、权限、家庭成员账号。
- 云同步、手机端、网页端。
- 条码扫描、OCR 识别、图片识别。
- 自动对接购物平台。
- 预算、价格、账单和财务分析。
- 复杂机器学习预测。

## 3. 核心概念

### 3.1 通用物资 Item

通用物资是统计和消耗的主口径，例如“面粉”“饭团”“沐浴露”“保鲜膜”。不同品牌、不同包装、不同批次的同类东西，都归属于同一个通用物资。

示例：

- “A 品牌高筋面粉 500g”和“B 品牌面粉 1000g”都属于“面粉”。
- “某品牌柠檬香沐浴露 750ml”和“另一品牌沐浴露 500ml”都属于“沐浴露”。

### 3.2 库存批次 StockBatch

库存批次是实际库存的记录单位。每次购买、入库或拆分管理都可以形成一个批次。批次记录品牌、包装信息、数量、单位、日期、有效期、位置等信息。

库存统计时，系统按通用物资汇总多个批次。

### 3.3 消耗单 ConsumptionRecord

消耗单是所有消耗行为的记录单位。它既可以表示一次性消耗，也可以表示持续消耗。

持续消耗单用于表达“已开封/消耗中”状态，例如：

- 一瓶沐浴露开始使用，创建消耗中记录。
- 一包面粉打开后持续使用，创建消耗中记录。
- 用完时完成消耗单，结算最终消耗量。

### 3.4 位置 Location

位置用于描述物资存放在哪里。第一版保持轻量，支持简单层级或路径式表达。

示例：

- 冰箱 / 冷藏层
- 冰箱 / 冷冻层
- 阳台柜子 / 上层
- 书房架子 / 第 2 层

## 4. 典型用户流程

### 4.1 新物资入库

1. 用户选择或新建通用物资。
2. 用户选择分类。
3. 用户填写品牌、数量、单位和包装信息。
4. 如果适用，填写生产日期、有效期。
5. 用户选择存放位置。
6. 系统生成库存批次。

### 4.2 一次性消耗

1. 用户选择要消耗的物资或库存批次。
2. 输入消耗数量。
3. 系统创建已完成消耗单。
4. 系统扣减对应批次库存。
5. 如果批次剩余量为 0，批次状态变为耗尽。

### 4.3 持续消耗 / 已开封

1. 用户选择某个库存批次。
2. 创建消耗中记录。
3. 系统在“正在消耗”统计中展示该物资。
4. 用户可在过程中补记消耗量或直接完成。
5. 完成时系统扣减最终消耗量，并保留消耗历史。

### 4.4 查看统计和预测

1. 用户打开统计页面。
2. 系统展示当前库存、已开封物资、临期物资、过期物资。
3. 系统根据历史消耗记录估算可用天数。
4. 对预计不足或临期风险物资给出提示。

## 6. 非功能需求

### 6.1 数据存储

- 使用本地关系型数据库，默认按 SQLite 思路设计。
- 数据必须可本地备份和迁移。
- 业务数据不依赖网络。

### 6.2 桌面端形态

- 使用 C/S 桌面端。
- 第一版单机使用，不需要服务端账号系统。
- 界面优先保证录入和查询效率。

### 6.3 易用性

- 高频操作应尽量少输入。
- 常用分类、单位、位置应提供默认值。
- 用户可随时新增分类、单位、物资和位置。
- 对数量、单位、日期等关键字段提供校验。

### 6.4 数据安全

- 不做远程传输。
- 应支持数据库文件备份。
- 后续可扩展导入/导出能力。

## 7. 第一版成功标准

- 用户可以完整录入家庭常见物资库存。
- 同一种物资的不同品牌、包装和批次可以被统一统计。
- 用户可以记录一次性消耗、部分消耗和持续消耗。
- 用户可以查看正在消耗中的物资和数量。
- 用户可以按分类、物资、位置查看库存。
- 系统可以识别临期、过期和耗尽批次。
- 系统可以基于历史消耗给出简单可用天数和补货建议。
