# 位置管理需求

## 1. 模块目标

本模块用于管理家庭物资的存放位置。位置管理保持轻量，重点是让用户能快速记录和查找物资在哪里。

核心目标：

- 支持常见家庭位置。
- 支持简单层级，例如冰箱 / 冷冻层。
- 支持按位置查看库存。
- 支持位置停用和历史保留。

## 2. 位置结构

第一版建议使用父子层级表达位置，同时在界面中展示为路径。

示例：

- 冰箱
- 冰箱 / 冷藏层
- 冰箱 / 冷冻层
- 阳台柜子
- 阳台柜子 / 上层
- 书房架子
- 书房架子 / 第 2 层
- 厨房柜子 / 调料区

## 3. 默认位置

系统可提供默认位置，用户可编辑或停用。

默认位置建议：

- 冰箱 / 冷藏层
- 冰箱 / 冷冻层
- 厨房柜子
- 厨房台面
- 阳台柜子
- 书房架子
- 卫生间
- 储物间
- 玄关柜
- 其他

## 4. 位置字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 名称 | 是 | 如冷冻层、第 2 层 |
| 父级位置 | 否 | 用于形成层级 |
| 路径名称 | 自动 | 如冰箱 / 冷冻层 |
| 排序 | 否 | 用于界面展示 |
| 状态 | 是 | 正常、停用 |
| 备注 | 否 | 用户补充说明 |

## 5. 位置功能

用户可以：

- 新增位置。
- 编辑位置名称。
- 调整父级位置。
- 设置排序。
- 停用位置。
- 查看位置下库存。

删除或停用规则：

- 已有关联库存或历史记录的位置不建议物理删除。
- 可以停用，停用后新增库存时默认不再展示。
- 历史批次仍显示原位置。

## 6. 库存位置规则

- 一个库存批次同一时间只对应一个主要位置。
- 如需拆分到多个位置，应拆成多个库存批次。
- 修改批次位置只影响当前库存记录，不改变历史消耗记录。
- 消耗记录应冗余保存消耗发生时的位置路径，便于历史查询。

## 7. 按位置查询

位置详情页应展示：

- 该位置及子位置的库存。
- 每个物资的当前剩余量。
- 临期和过期批次。
- 正在消耗中的批次。

位置统计应支持：

- 只看当前层级。
- 包含子位置汇总。

示例：

- 查看“冰箱”时，可汇总冷藏层和冷冻层。
- 查看“冰箱 / 冷冻层”时，只显示冷冻层库存。

## 8. 位置选择交互

库存录入时，位置选择应支持：

- 从常用位置中选择。
- 输入关键字搜索。
- 快速新建位置。
- 为空，表示暂未记录位置。

为了录入效率，系统可以记住最近使用的位置。

## 9. 验收场景

- 用户可以新增“书房架子 / 第 2 层”。
- 用户录入库存时可以选择“冰箱 / 冷冻层”。
- 用户查看“冰箱”时能看到冷藏层和冷冻层的汇总库存。
- 用户停用“旧柜子”后，新增库存不再默认展示该位置，但历史记录仍能看到。
- 一个库存批次移动到新位置后，库存列表展示新位置。
