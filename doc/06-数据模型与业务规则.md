# 数据模型与业务规则

## 1. 设计原则

系统采用本地关系型数据库设计，默认可用 SQLite 实现。需求文档不绑定具体开发语言或 UI 框架。

核心设计原则：

- 通用物资负责统一统计口径。
- 库存批次负责真实库存和差异化信息。
- 消耗单负责所有消耗流水和已开封状态。
- 分类、单位、位置都允许用户自定义。
- 历史记录应尽量保留，不随基础资料修改而丢失。

## 2. 核心实体

### 2.1 Category 分类

用途：管理物资分类。

字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| name | 分类名称 |
| parent_id | 父级分类，可为空 |
| default_expiry_remind_days | 默认临期提醒天数 |
| sort_order | 排序 |
| status | 正常、停用 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

### 2.2 Item 通用物资

用途：作为库存统计和消耗统计的统一口径。

字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| name | 物资名称 |
| category_id | 分类 |
| default_unit_id | 默认单位 |
| enable_expiry | 是否启用有效期 |
| expiry_remind_days | 自定义临期提醒天数 |
| note | 备注 |
| status | 正常、停用 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

唯一性建议：

- 同一分类下物资名称不重复。
- 不同分类下是否允许重名可由系统配置决定，第一版建议允许但提示用户。

### 2.3 Unit 单位

用途：管理数量单位和换算关系。

字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| name | 单位名称，如 g、kg、个 |
| unit_type | 数量、重量、容量、长度等 |
| base_unit_id | 基础单位，可为空 |
| conversion_rate_to_base | 换算到基础单位的倍率 |
| status | 正常、停用 |

示例：

- g 的基础单位是 g，倍率 1。
- kg 的基础单位是 g，倍率 1000。
- ml 的基础单位是 ml，倍率 1。
- L 的基础单位是 ml，倍率 1000。

### 2.4 Location 位置

用途：管理存放位置。

字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| name | 位置名称 |
| parent_id | 父级位置，可为空 |
| path_name | 完整路径名称 |
| sort_order | 排序 |
| status | 正常、停用 |
| note | 备注 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

### 2.5 StockBatch 库存批次

用途：记录实际库存。

字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| item_id | 通用物资 |
| brand | 品牌 |
| spec | 规格文本 |
| initial_quantity | 初始数量 |
| current_quantity | 当前剩余量 |
| unit_id | 单位 |
| production_date | 生产日期 |
| expiry_date | 有效期至 |
| inbound_date | 入库日期 |
| location_id | 存放位置 |
| status | 正常、耗尽、过期、废弃 |
| note | 备注 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

业务约束：

- initial_quantity 必须大于 0。
- current_quantity 不能小于 0。
- current_quantity 不能大于 initial_quantity，除非后续引入库存调整单。
- unit_id 必须与消耗单单位兼容。

### 2.6 ConsumptionRecord 消耗单

用途：记录一次性、部分或持续消耗。

字段建议：

| 字段 | 说明 |
| --- | --- |
| id | 主键 |
| item_id | 通用物资 |
| stock_batch_id | 库存批次 |
| consumption_type | 一次性、部分、持续 |
| status | 草稿、消耗中、已完成、已取消 |
| planned_quantity | 计划消耗量 |
| actual_quantity | 实际消耗量 |
| unit_id | 单位 |
| started_at | 开始时间 |
| completed_at | 完成时间 |
| location_snapshot | 消耗时位置路径 |
| note | 备注 |
| created_at | 创建时间 |
| updated_at | 更新时间 |

业务约束：

- 已完成记录必须有 actual_quantity。
- actual_quantity 必须大于 0。
- 已完成记录必须有 completed_at。
- 已取消记录不参与库存扣减、统计和预测。

## 3. 状态枚举

### 3.1 基础资料状态

```text
NORMAL = 正常
DISABLED = 停用
```

适用于：

- Category
- Item
- Unit
- Location

### 3.2 库存批次状态

```text
NORMAL = 正常
DEPLETED = 耗尽
EXPIRED = 过期
DISCARDED = 废弃
```

### 3.3 消耗单状态

```text
DRAFT = 草稿
IN_PROGRESS = 消耗中
COMPLETED = 已完成
CANCELLED = 已取消
```

### 3.4 消耗类型

```text
ONCE = 一次性
PARTIAL = 部分
CONTINUOUS = 持续
```

## 4. 关键业务规则

### 4.1 入库规则

- 新增库存批次时，初始数量和当前剩余量默认相同。
- 如果填写生产日期和保质期天数，可计算有效期至。
- 有效期为空时，该批次不参与临期/过期判断。
- 入库日期默认当前日期。

### 4.2 消耗扣减规则

- 只有已完成消耗单会扣减库存。
- 草稿、消耗中、已取消不扣减库存。
- 完成消耗单时，系统校验批次剩余量是否足够。
- 扣减后剩余量为 0 时，批次状态变为耗尽。

### 4.3 已开封规则

- 存在消耗中的消耗单，即视为对应批次已开封。
- 已开封不是库存批次主状态。
- 一个批次原则上允许存在一个未完成的持续消耗单。
- 如果确需多个消耗中记录，系统应提示用户确认。

### 4.4 临期和过期规则

- 当前日期大于有效期至，则视为过期。
- 当前日期距离有效期至小于等于提醒天数，则视为临期。
- 过期批次可自动标记，也可仅在查询中计算展示；第一版建议计算展示，用户确认后再变更状态。

### 4.5 单位换算规则

- 同类型单位可以换算。
- 不同类型单位不能换算。
- 批次和消耗单单位不一致时，必须能换算到同一基础单位。
- 汇总统计优先展示通用物资默认单位。

### 4.6 历史保留规则

- 分类、物资、单位、位置被停用后，历史数据仍可查询。
- 消耗记录应保留创建时的关键信息快照，例如位置路径。
- 不建议物理删除已有业务记录。

## 5. 数据一致性

系统应保证：

- 库存剩余量不能为负数。
- 已完成消耗记录和库存扣减保持一致。
- 已取消消耗记录不影响库存。
- 耗尽、废弃、过期批次默认不参与可用库存预测。
- 基础资料停用不影响历史记录展示。

## 6. MVP 数据表建议

第一版至少需要以下表：

- categories
- items
- units
- locations
- stock_batches
- consumption_records

可选扩展表：

- stock_adjustments：库存调整单。
- app_settings：系统设置。
- import_export_logs：导入导出日志。

## 7. 验收场景

- 同一种物资的多个批次可以按 item_id 汇总。
- 已完成消耗单会扣减对应 stock_batch 的 current_quantity。
- 消耗中记录不会立即扣减库存，但会让批次被识别为已开封。
- 单位 g 和 kg 可以汇总，g 和 个不能汇总。
- 停用分类后，历史库存和消耗记录仍能正常显示。
