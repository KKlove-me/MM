# 物资分类与基础资料需求

## 1. 模块目标

本模块用于定义家庭物资的基础资料，包括分类、通用物资、单位和包装口径。它为库存录入、消耗统计和预测提供统一口径。

核心目标：

- 让用户可以把家庭常用物资按分类管理。
- 让不同品牌、包装、批次的同类物资归并到同一个通用物资。
- 让数量单位和包装信息尽量清晰，便于后续库存扣减和统计。

## 2. 分类管理

### 2.1 分类结构

分类支持两级或多级结构。第一版建议支持父子分类，但界面上保持简单。

默认一级分类：

- 食品
- 日用
- 园艺
- 药品/护理
- 工具/耗材
- 其他

食品默认子类示例：

- 干货
- 冷冻
- 冷藏
- 半成品
- 即食
- 调料
- 饮品

日用默认子类示例：

- 厨房耗材
- 洗护用品
- 清洁用品
- 纸品
- 收纳用品

园艺默认子类示例：

- 肥料
- 种子
- 土壤/基质
- 工具耗材

### 2.2 分类功能

用户可以：

- 新增分类。
- 编辑分类名称。
- 调整父级分类。
- 停用分类。
- 查看分类下的物资数量。

删除分类的规则：

- 如果分类下已有物资，不允许直接删除。
- 可改为停用，历史数据继续保留。
- 若确需删除，必须先迁移分类下物资。

## 3. 通用物资管理

### 3.1 通用物资定义

通用物资是库存统计和消耗统计的核心对象。

字段建议：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 名称 | 是 | 如面粉、饭团、沐浴露 |
| 分类 | 是 | 关联分类 |
| 默认单位 | 是 | 如个、g、ml |
| 默认临期提醒天数 | 否 | 不填时使用系统默认值 |
| 是否启用有效期 | 否 | 食品通常启用，部分日用品可不启用 |
| 备注 | 否 | 用户自定义说明 |
| 状态 | 是 | 正常、停用 |

### 3.2 物资合并口径

同一种东西即使品牌和包装不同，也应归为同一个通用物资。

示例：

| 通用物资 | 实际库存批次 |
| --- | --- |
| 面粉 | A 品牌高筋面粉 500g、B 品牌普通面粉 1000g |
| 保鲜袋 | 中号保鲜袋 100 只、大号保鲜袋 50 只 |
| 沐浴露 | 柠檬香 750ml、无香型 500ml |

系统在统计时默认按通用物资汇总，也允许用户展开查看具体批次。

### 3.3 物资新增和编辑

用户可以：

- 新增通用物资。
- 修改名称、分类、默认单位、提醒设置。
- 停用不再使用的物资。
- 查看该物资关联的所有库存批次和消耗记录。

修改规则：

- 修改通用物资名称不影响历史批次和消耗记录。
- 修改默认单位只影响之后新建的库存批次，不自动换算历史数据。
- 已有关联数据的物资不建议物理删除。

## 4. 单位管理

### 4.1 单位类型

单位按类型分组，第一版只保留最常用的基础单位。

| 类型 | 单位示例 | 说明 |
| --- | --- | --- |
| 数量 | 个 | 计数单位 |
| 重量 | g | 重量单位 |
| 容量 | ml | 容量单位 |
| 长度 | cm | 长度单位 |

### 4.2 单位换算规则

第一版不做单位自动换算：

- 批次库存单位、单包装内容单位和消耗单位需要保持一致。
- 不支持 kg 与 g、L 与 ml、m 与 cm 之间的自动换算。
- 包装数量只是辅助录入和展示，不作为内容物单位参与换算。

### 4.3 默认单位建议

| 物资 | 默认单位 |
| --- | --- |
| 饭团 | 个 |
| 面粉 | g |
| 大米 | g |
| 沐浴露 | ml |
| 洗发水 | ml |
| 香皂 | 个 |
| 保鲜袋 | 个 |
| 保鲜膜 | cm |
| 肥料 | g |
| 种子 | 个 |

## 5. 包装管理

包装信息用于描述购买包装或实际形态。对于面粉、沐浴露、肥料等可部分消耗物资，系统应使用结构化包装字段，把“包装数量”和“每包装内容量”换算成库存真实总量。

示例：

- 500g/袋
- 750ml/瓶
- 100 只/盒
- 30cm x 50m/卷

需求规则：

- 库存真实扣减口径始终是内容物总量，例如 g、ml、只、m。
- 包装字段用于辅助录入和消耗，例如“1000g/袋 × 5 袋”自动保存为“5000g”库存。
- 不再保存单独的包装描述文本；列表展示需要的“1000g/袋”来自结构化包装字段。
- 有包装信息的批次，列表中应同时展示内容物剩余量和折算包装数量，例如“3500g，约 3.5 袋”。
- 无包装信息的旧批次或计数型物资，继续按原有“数量 + 单位”展示和扣减。

结构化包装字段建议：

| 字段 | 说明 |
| --- | --- |
| 包装数量 | 本次录入多少袋、瓶、包、盒 |
| 包装单位 | 袋、瓶、包、盒等计数单位 |
| 单包装内容量 | 每个包装内含多少内容物 |
| 内容单位 | 个、g、ml、cm |
| 库存单位 | 批次最终保存和扣减使用的单位 |

示例：

| 录入 | 库存真实总量 | 展示 |
| --- | --- | --- |
| 1000g/袋 × 5 袋 | 5000g | 5000g，约 5 袋 |
| 750ml/瓶 × 2 瓶 | 1500ml | 1500ml，约 2 瓶 |
| 100 只/盒 × 3 盒 | 300只 | 300只，约 3 盒 |

## 6. 页面和交互需求

### 6.1 分类列表

应展示：

- 分类名称。
- 父级分类。
- 关联物资数量。
- 状态。

支持：

- 新增、编辑、停用。
- 按名称搜索。

### 6.2 物资列表

应展示：

- 物资名称。
- 分类。
- 默认单位。
- 当前库存汇总。
- 是否有临期/过期批次。
- 状态。

支持：

- 新增、编辑、停用。
- 按名称搜索。
- 按分类筛选。

### 6.3 物资详情

应展示：

- 基础资料。
- 当前库存批次。
- 最近消耗记录。
- 预测可用天数。

## 7. 验收场景

- 用户可以新增“食品 / 干货”分类。
- 用户可以新增“面粉”物资，默认单位为 g。
- 用户可以把不同品牌、包装的面粉批次归并到“面粉”。
- 用户修改“面粉”的分类后，历史库存和消耗记录仍然可查询。
- 已有关联库存的分类无法被直接删除。
