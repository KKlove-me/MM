# 库存录入与批次管理需求

## 1. 模块目标

本模块用于记录家庭物资的实际库存。库存采用批次管理，同一种通用物资可以有多个库存批次，每个批次记录品牌、规格、包装信息、真实库存总量、生产日期、有效期和存放位置。

核心目标：

- 支持快速录入家庭物资。
- 支持同类物资不同品牌、规格和日期的精细管理。
- 支持数量型、重量型、容量型等多种库存。
- 支持后续消耗、临期提醒和统计预测。

## 2. 库存批次定义

库存批次是实际库存的最小管理单位。

示例：

- A 品牌面粉，500g/袋，1 袋，剩余 500g，放在厨房柜子。
- B 品牌面粉，1kg/袋，5 袋，剩余 5000g，约 5 袋，放在阳台柜子。
- 饭团，10 个，放在冰箱冷冻层。
- 沐浴露，750ml/瓶，剩余 750ml，放在浴室。

## 3. 批次字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 通用物资 | 是 | 关联 Item |
| 分类 | 是 | 默认来自通用物资，可在录入时展示 |
| 品牌 | 否 | 可为空 |
| 规格 | 否 | 如 500g/袋、750ml/瓶 |
| 初始数量 | 是 | 入库时内容物总量 |
| 当前剩余量 | 是 | 当前可用内容物总量 |
| 单位 | 是 | 内容物单位，如个、g、ml、m |
| 包装数量 | 否 | 如 5 袋、2 瓶中的数量 |
| 包装单位 | 否 | 袋、瓶、包、盒等 |
| 单包装内容量 | 否 | 如每袋 1000g 中的 1000 |
| 单包装内容单位 | 否 | g、kg、ml、L、只、m 等 |
| 生产日期 | 否 | 食品类建议填写 |
| 有效期至 | 否 | 可直接填写日期 |
| 保质期天数 | 否 | 可由生产日期 + 保质期计算有效期 |
| 入库日期 | 是 | 默认当前日期 |
| 存放位置 | 否 | 关联 Location |
| 状态 | 是 | 正常、耗尽、过期、废弃 |
| 备注 | 否 | 用户补充说明 |

## 4. 录入方式

### 4.1 新增库存批次

用户可以从库存页面新增批次：

1. 选择或新建通用物资。
2. 自动带出分类和默认单位。
3. 填写品牌、规格。
4. 选择录入方式：按包装录入或直接填总量。
5. 按包装录入时，填写单包装内容量、内容单位、包装数量、包装单位，系统自动计算库存总量。
6. 直接填总量时，填写总量和单位。
7. 填写生产日期、有效期或保质期。
8. 选择存放位置。
9. 保存后生成库存批次。

### 4.2 从物资详情新增

用户在某个通用物资详情页新增库存时：

- 通用物资自动填入。
- 分类和默认单位自动填入。
- 用户只需填写实际批次信息。

### 4.3 快速录入

第一版可提供简化表单：

- 物资名称。
- 总量或包装数量。
- 单位。
- 位置。
- 有效期至。

其余字段可为空或使用默认值。

### 4.4 按包装录入

适用于“每袋 1000g，买了 5 袋”这类场景。

录入字段：

- 每包装内容量：如 1000。
- 内容单位：如 g。
- 包装数量：如 5。
- 包装单位：如袋。
- 库存单位：如 g。

系统行为：

- 自动计算库存真实总量：每包装内容量 × 包装数量，并换算到库存单位。
- 保存规格文本；如果用户未填写规格文本，可自动生成“1000g/袋”。
- 批次列表显示真实剩余量，并折算展示约多少包装。

示例：

| 录入 | 保存库存 | 批次展示 |
| --- | --- | --- |
| 1000g/袋 × 5 袋 | initial_quantity = 5000，unit = g | 剩余 5000g，约 5 袋 |
| 1kg/袋 × 2 袋，库存单位 g | initial_quantity = 2000，unit = g | 剩余 2000g，约 2 袋 |

### 4.5 直接填总量

适用于没有包装换算需求的物资，或用户只想记录总量的旧方式。

示例：

- 饭团 10 个。
- 面粉 500g。
- 保鲜袋 100 只。

系统行为：

- 只保存真实总量和单位。
- 不保存包装数量、包装单位和单包装内容量。
- 消耗时只能按内容量或同类单位扣减，不能按包装快捷扣减。

## 5. 数量和单位规则

### 5.1 数量型库存

适用于按个、只、块、瓶、包等计数的物资。

示例：

- 饭团 10 个。
- 香皂 3 块。
- 保鲜袋 100 只。

消耗时按同单位扣减。

### 5.2 重量型库存

适用于面粉、大米、肥料等。

示例：

- 面粉 500g。
- 大米 5kg。
- 肥料 1000g。

系统支持 g 和 kg 换算，但库存批次内部建议保存为基础单位 g。

### 5.3 容量型库存

适用于沐浴露、洗发水、饮料、清洁剂等。

示例：

- 沐浴露 750ml。
- 洗发水 500ml。
- 牛奶 1L。

系统支持 ml 和 L 换算，但库存批次内部建议保存为基础单位 ml。

### 5.4 包装与真实库存总量

包装单位和库存单位不是同一层概念：

- 包装单位表示外部包装数量，如袋、瓶、盒。
- 库存单位表示真实可消耗内容物，如 g、ml、只。
- 系统保存和扣减库存时以库存单位为准。
- 包装单位只有在批次记录了“单包装内容量”时，才能换算为库存单位。

示例：

- 面粉“1000g/袋 × 5 袋”真实库存是 5000g。
- 消耗 1 袋时，系统扣减 1000g。
- 消耗 500g 时，系统扣减 500g，剩余展示为“4500g，约 4.5 袋”。

## 6. 日期和有效期

### 6.1 日期录入

库存批次可记录：

- 生产日期。
- 有效期至。
- 保质期天数。
- 入库日期。

有效期计算规则：

- 如果用户直接填写有效期至，则以该日期为准。
- 如果用户填写生产日期和保质期天数，则系统计算有效期至。
- 如果两者都填写，系统应提示用户确认，默认以有效期至为准。

### 6.2 临期判断

临期提醒天数优先级：

1. 通用物资自定义提醒天数。
2. 分类默认提醒天数。
3. 系统默认提醒天数。

临期状态：

- 当前日期晚于有效期至：过期。
- 当前日期距离有效期至小于等于提醒天数：临期。
- 无有效期：不参与临期判断。

## 7. 批次状态

| 状态 | 含义 | 触发方式 |
| --- | --- | --- |
| 正常 | 可用库存 | 新增后默认状态 |
| 耗尽 | 当前剩余量为 0 | 消耗完成后自动触发 |
| 过期 | 已超过有效期 | 系统计算或用户确认 |
| 废弃 | 用户主动标记不用 | 手动操作 |

状态规则：

- 耗尽批次不再参与可用库存统计。
- 废弃批次不再参与可用库存统计，但保留历史。
- 过期批次应在库存中提示，默认不建议参与可用天数预测。
- 当前剩余量不能小于 0。

## 8. 查询和筛选

库存列表应支持：

- 按物资名称搜索。
- 按分类筛选。
- 按位置筛选。
- 按状态筛选。
- 按临期/过期筛选。
- 按入库日期或有效期排序。

库存列表应展示：

- 通用物资名称。
- 品牌。
- 规格。
- 当前剩余量和单位；有包装信息时同时展示约多少包装。
- 存放位置。
- 有效期。
- 状态。

## 9. 验收场景

- 用户录入 500g 面粉，系统生成一个库存批次。
- 用户再次录入另一品牌 1kg 面粉，统计时两批次合并为面粉总库存 1500g。
- 用户录入“1000g/袋 × 5 袋”面粉，系统保存为 5000g 库存，并展示“5000g，约 5 袋”。
- 用户消耗该面粉 1 袋后，库存剩余 4000g，展示“4000g，约 4 袋”。
- 用户继续消耗该面粉 500g 后，库存剩余 3500g，展示“3500g，约 3.5 袋”。
- 用户录入饭团 10 个，消耗 1 个后剩余 9 个。
- 用户录入有效期为 7 天后的食品，系统在临期列表中显示。
- 用户将某批次标记为废弃后，该批次不参与可用库存统计。
