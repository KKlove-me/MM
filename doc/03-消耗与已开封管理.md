# 消耗与已开封管理需求

## 1. 模块目标

本模块用于记录物资消耗行为。系统采用独立消耗单作为核心模型，所有一次性消耗、部分消耗和持续消耗都通过消耗单记录。

核心目标：

- 记录每次消耗发生了什么、消耗了多少、来自哪个库存批次。
- 支持一次性扣减和部分扣减。
- 支持“消耗中”状态，用于表达已开封或持续使用。
- 为统计分析和预测提供历史消耗数据。

## 2. 消耗单类型

### 2.1 一次性消耗

一次性消耗表示创建后立即完成的消耗。

示例：

- 吃掉 1 个饭团。
- 用掉 250g 面粉。
- 用掉 1 袋已记录包装规格的面粉。
- 用掉 1 块香皂。

保存后：

- 消耗单状态为已完成。
- 库存批次当前剩余量立即扣减。
- 批次剩余量为 0 时，状态变为耗尽。

### 2.2 部分消耗

部分消耗表示从一个批次中扣减一部分数量。

示例：

- 一包 500g 面粉，本次使用 250g。
- 一批 1000g/袋、共 5 袋的面粉，本次使用 500g。
- 一瓶 750ml 沐浴露，本次估算使用 30ml。

部分消耗可以是一次性完成，也可以先进入消耗中状态。

### 2.3 持续消耗

持续消耗表示某个物资已开封或正在使用，但尚未最终结算。

示例：

- 一瓶沐浴露开始使用。
- 一包面粉打开后放在厨房继续使用。
- 一袋肥料开始使用，但还没用完。

创建持续消耗单后：

- 消耗单状态为消耗中。
- 该物资出现在“正在消耗”统计中。
- 库存可以选择暂不扣减，或先记录预计消耗量；第一版默认完成时结算。

## 3. 消耗单字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 通用物资 | 是 | 冗余保存，便于统计 |
| 库存批次 | 是 | 消耗来源 |
| 消耗类型 | 是 | 一次性、部分、持续 |
| 状态 | 是 | 草稿、消耗中、已完成、已取消 |
| 计划消耗量 | 否 | 持续消耗时可填写 |
| 实际消耗量 | 否 | 完成时填写，按内容物单位保存 |
| 单位 | 是 | 内容物单位，应与批次单位兼容 |
| 包装消耗量 | 否 | 按包装消耗时填写，如 1 袋 |
| 包装单位 | 否 | 袋、瓶、包、盒等，来自库存批次包装信息 |
| 开始时间 | 是 | 默认当前时间 |
| 完成时间 | 否 | 完成时填写 |
| 用途/备注 | 否 | 如做饭、日常洗护 |

## 4. 状态流转

消耗单状态：

- 草稿：已记录但不生效。
- 消耗中：正在使用或已开封。
- 已完成：已结算并扣减库存。
- 已取消：不再生效。

状态流转规则：

- 草稿可以变为消耗中、已完成或已取消。
- 消耗中可以变为已完成或已取消。
- 已完成不能再取消；如需修正，应通过编辑记录或反向调整处理。
- 已取消不参与统计和预测。

## 5. 库存扣减规则

### 5.1 一次性消耗扣减

保存已完成消耗单时立即扣减库存。

规则：

- 实际消耗量必须大于 0。
- 实际消耗量不能超过批次当前剩余量。
- 单位必须与库存批次单位相同或可换算。
- 如果按包装消耗，系统先根据批次的单包装内容量换算为内容物消耗量，再扣减库存。

示例：

- 面粉批次为“1000g/袋 × 5 袋”，当前剩余 5000g。
- 用户选择按包装消耗 1 袋。
- 系统保存实际消耗量为 1000g，同时记录包装消耗量为 1 袋。
- 库存剩余 4000g，展示“4000g，约 4 袋”。

### 5.2 部分内容量消耗

部分内容量消耗用于“没有用完一个包装”的场景。

规则：

- 用户选择按内容量消耗。
- 填写内容物数量和单位，例如 500g。
- 系统按内容物数量扣减库存。
- 有包装信息的批次，扣减后继续折算展示剩余约多少包装。

示例：

- 面粉批次当前剩余 4000g，包装规格为 1000g/袋。
- 用户本次消耗 500g。
- 系统扣减 500g，剩余 3500g。
- 批次展示为“3500g，约 3.5 袋”。

### 5.3 持续消耗扣减

第一版默认采用完成时扣减：

- 创建消耗中记录时，不扣减批次当前剩余量。
- 完成消耗单时，用户填写实际消耗量。
- 系统扣减库存。

这样可以避免长期使用物资无法准确估算时造成库存错误。

### 5.4 完全用完

完成消耗单时，用户可选择“用完该批次”。

系统行为：

- 实际消耗量自动填为批次当前剩余量。
- 批次当前剩余量变为 0。
- 批次状态变为耗尽。

## 6. 已开封表达

“已开封”不作为库存批次的独立主状态，而通过消耗中的消耗单表达。

判断规则：

- 某库存批次存在状态为消耗中的消耗单，则该批次视为已开封。
- 某通用物资存在任一消耗中的消耗单，则该物资出现在正在消耗统计中。

展示内容：

- 物资名称。
- 批次品牌和规格。
- 真实剩余量和折算包装数量。
- 位置。
- 开始消耗时间。
- 已持续天数。
- 计划或估计消耗量。

## 7. 消耗选择批次规则

用户手动选择批次为主，系统可提供建议排序：

1. 已开封/消耗中的批次优先。
2. 临期批次优先。
3. 有效期较早的批次优先。
4. 入库较早的批次优先。

系统不得在用户不确认的情况下自动跨批次扣减。

## 8. 查询和统计

消耗记录列表应支持：

- 按物资搜索。
- 按分类筛选。
- 按状态筛选。
- 按时间范围筛选。
- 按位置筛选。

消耗记录应展示：

- 物资名称。
- 批次信息。
- 消耗量；按包装消耗时同时展示包装数量。
- 状态。
- 开始时间。
- 完成时间。
- 备注。

## 9. 验收场景

- 用户消耗 1 个饭团，系统创建已完成消耗单并将库存从 10 个扣为 9 个。
- 用户消耗 250g 面粉，系统将 500g 批次扣为 250g。
- 用户从“1000g/袋 × 5 袋”的面粉批次中消耗 1 袋，系统扣减 1000g，剩余 4000g。
- 用户从同一批次中部分消耗 500g，系统扣减 500g，剩余 3500g，并展示约 3.5 袋。
- 用户将沐浴露创建为消耗中，库存暂不扣减，但统计页显示正在消耗。
- 用户完成沐浴露消耗单并选择用完该批次，库存变为 0，批次状态为耗尽。
- 用户取消消耗中记录后，该物资不再出现在正在消耗统计中。
